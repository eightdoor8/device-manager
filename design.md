# Device Manager - モバイルアプリデザイン計画書

## アプリ概要
社内QAチーム向けのモバイル端末管理アプリ。端末の自動登録、貸出管理、検索機能を提供し、新人QAエンジニアの教育題材としても使用する。

## デザイン方針
- **プラットフォーム**: iOS/Android両対応（React Native/Expo）
- **デザインシステム**: Material 3ベースのシンプルで見やすいデザイン
- **画面向き**: ポートレート（9:16）
- **操作性**: 片手操作を考慮した配置

## カラーパレット
- **プライマリ**: `#2196F3` (青 - 信頼性と技術を表現)
- **セカンダリ**: `#4CAF50` (緑 - 利用可能状態を表現)
- **エラー**: `#F44336` (赤 - 貸出中や警告を表現)
- **背景**: `#FFFFFF` (ライトモード), `#121212` (ダークモード)
- **テキスト**: `#212121` (プライマリ), `#757575` (セカンダリ), `#BDBDBD` (無効)

## タイポグラフィ
- **タイトル**: 28pt, Bold, lineHeight 36
- **サブタイトル**: 20pt, SemiBold, lineHeight 28
- **ボディ**: 16pt, Regular, lineHeight 24
- **キャプション**: 14pt, Regular, lineHeight 20
- **ラベル**: 12pt, Medium, lineHeight 16

## スペーシング
- 8ptグリッドシステムを採用
- 基本単位: 8, 12, 16, 24, 32

## 画面一覧と機能

### 1. ログイン画面 (Login)
**目的**: Firebase Authenticationによるメールアドレス・パスワード認証

**コンテンツ**:
- アプリロゴ
- メールアドレス入力フィールド
- パスワード入力フィールド
- ログインボタン
- エラーメッセージ表示エリア

**レイアウト**:
- 中央配置のフォーム
- ボタンは画面下部1/3（親指ゾーン）に配置
- 入力フィールドは最小44pt高さ

### 2. 端末一覧画面 (Dashboard)
**目的**: 登録済み端末の一覧表示と検索・フィルタ

**コンテンツ**:
- 検索バー（上部固定）
- フィルタボタン（OS、メーカー、ステータス）
- 端末カードリスト（FlatList使用）
  - 機種名（例: iPhone 14 Pro Max）
  - OSバージョン（例: iOS 17.2）
  - 画面サイズ（例: 6.7インチ）
  - ステータスバッジ（利用可/貸出中）
  - 利用者名（貸出中の場合）
- フローティングアクションボタン（右下）: 「この端末を登録」

**レイアウト**:
- 検索バーは上部セーフエリア直下
- カードは16ptマージン、12pt角丸
- ステータスバッジは右上に配置
- FABは右下、セーフエリアから16pt

### 3. 端末詳細画面 (Device Detail)
**目的**: 端末の詳細情報表示と貸出・返却操作

**コンテンツ**:
- 端末情報セクション
  - 機種名（大見出し）
  - OS名・バージョン
  - 内部モデルID
  - 物理メモリ
  - UUID
  - 画面サイズ
- ステータスセクション
  - 現在のステータス（大きく表示）
  - 利用者名（貸出中の場合）
  - 貸出日時（貸出中の場合）
- アクションボタン
  - 「借りる」ボタン（利用可の場合）
  - 「返す」ボタン（自分が借りている場合）
  - 編集ボタン（管理者のみ）

**レイアウト**:
- スクロール可能な縦レイアウト
- セクションごとに16pt間隔
- アクションボタンは画面下部固定（セーフエリア考慮）

### 4. 端末登録画面 (Register Device)
**目的**: 現在使用中の端末を自動検出して登録

**コンテンツ**:
- 自動検出情報表示
  - OS名・バージョン（自動取得）
  - 内部モデルID（自動取得）
  - 機種名（マッピング後、編集可能）
  - 物理メモリ（自動取得）
  - UUID（自動取得）
- 手動入力フィールド
  - 画面サイズ（例: 6.7インチ）
  - メモ（任意）
- 登録ボタン

**レイアウト**:
- フォーム形式
- 自動取得項目は薄いグレー背景で区別
- 編集可能項目は白背景
- 登録ボタンは画面下部固定

### 5. フィルタモーダル (Filter Modal)
**目的**: 端末一覧の絞り込み条件設定

**コンテンツ**:
- OSバージョン選択（チップ形式）
- メーカー選択（チップ形式）
- ステータス選択（利用可/貸出中/すべて）
- 適用ボタン
- クリアボタン

**レイアウト**:
- ボトムシート形式（画面下部から表示）
- 16-24pt角丸
- セーフエリア考慮

## 主要ユーザーフロー

### フロー1: 端末を借りる
1. ログイン画面でメール・パスワード入力 → ログイン
2. 端末一覧画面で目的の端末を検索/フィルタ
3. 端末カードをタップ → 端末詳細画面へ遷移
4. 「借りる」ボタンをタップ
5. 確認ダイアログ表示 → 「OK」をタップ
6. ステータスが「貸出中」に変更、自分の名前が表示される

### フロー2: 端末を登録する
1. ログイン後、端末一覧画面のFABをタップ
2. 端末登録画面へ遷移
3. 自動検出された情報を確認
4. 必要に応じて機種名を修正
5. 画面サイズなど手動項目を入力
6. 「登録」ボタンをタップ
7. Firestoreに保存 → 端末一覧画面へ戻る

### フロー3: 端末を返却する
1. 端末一覧画面で自分が借りている端末をタップ
2. 端末詳細画面で「返す」ボタンをタップ
3. 確認ダイアログ表示 → 「OK」をタップ
4. ステータスが「利用可」に変更

## ナビゲーション構造
- **タブナビゲーション**（ボトムタブ）
  - ホーム（端末一覧）
  - マイデバイス（自分が借りている端末）
  - プロフィール（ユーザー情報、ログアウト）
- **スタックナビゲーション**
  - 端末詳細
  - 端末登録
  - フィルタモーダル

## コンポーネント設計
- **DeviceCard**: 端末一覧のカードコンポーネント
- **StatusBadge**: ステータス表示バッジ
- **FilterChip**: フィルタ選択用チップ
- **DetailRow**: 詳細画面の情報行
- **PrimaryButton**: プライマリアクションボタン
- **SecondaryButton**: セカンダリアクションボタン

## アクセシビリティ
- すべてのタッチターゲットは最小44pt
- 十分なコントラスト比（WCAG AA準拠）
- スクリーンリーダー対応（適切なラベル付け）
- ダークモード完全対応
